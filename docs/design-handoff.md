# Handoff: "וינה לשלומי" — אפליקציית מתנת יום הולדת / מדריך טיול אינטראקטיבי

> **שפת המסמך:** עברית (קהל היעד), עם מונחים טכניים באנגלית.
> **מטרת המסמך:** לאפשר למפתח (למשל באמצעות Claude Code) לבנות את האפליקציה האמיתית מאפס, בלי שהשתתף בשיחת העיצוב.

---

## 1. סקירה (Overview)

"וינה לשלומי" היא אפליקציית מובייל שהיא **מתנת יום הולדת** — מדריך טיול אישי וחי לטיול בן 4 ימים בווינה והאלפים הווינאיים, עבור זוג (שלומי ויפעת). האפליקציה משלבת:

- ברכת יום הולדת אישית בכניסה + הקלטות קול של בת/בן הזוג ("הקול של יפעת")
- מפה חיה עם 11 אתרי המסלול, מקומות אוכל מקומיים, אטרקציות נוספות, ומיקום המשתמש
- מסלול יומי מפורט (שעות, הגעה, מחירים)
- תכנון אישי: טיסות, מלון, הסעה מהשדה, תזכורות והתראות
- מצב הליכה עם הכוונה רגלית
- **מנוע המלצות חי** — תוך כדי הליכה מזהה מקומות מומלצים ברדיוס 200 מ' ומציע אותם עם "עלות הסטייה" מהמסלול
- פינות "הידעת?" עם טריוויה היסטורית בכל אתר
- יומן מסע: דירוג מקומות, תיעוד, ציר זמן של היכן ביקרת
- תמונות + סרטוני "רגעים" לכל יום, וסרטון הקדשת יום הולדת בסוף הטיול

---

## 2. על קבצי העיצוב (About the Design Files)

הקבצים בחבילה הזו הם **רפרנס עיצובי שנבנה ב-HTML/CSS/JS** — פרוטוטייפ אינטראקטיבי שמדגים את המראה וההתנהגות הרצויים. **אין להעלות את ה-HTML הזה כמוצר נייטיב כפי שהוא.** המשימה היא **לשחזר את העיצובים והפונקציונליות הללו בסביבת פיתוח אמיתית** (ראו המלצת ה-stack בסעיף 9), תוך שימוש בדפוסים ובספריות המקובלים שם.

הפרוטוטייפ הוא **המוח המלא של האפליקציה** — כל המסכים, הזרימות, מבני הנתונים והלוגיקה כבר מוגדרים ועובדים בדפדפן. מה שחסר כדי שזו תהיה אפליקציה "מתפקדת באמת על הטלפון" הוא: מעטפת נייטיב, שרת (backend), וחיבור ל-APIs חיצוניים. כל אלה מפורטים בסעיפים 8–11.

## 3. רמת נאמנות (Fidelity)

**Hi-fi (נאמנות גבוהה).** הפרוטוטייפ הוא mockup כמעט פיקסל-מושלם: צבעים סופיים, טיפוגרפיה, מרווחים, אנימציות ואינטראקציות אמיתיות. המפתח צריך לשחזר את ה-UI באופן מדויק, תוך שימוש בספריות ה-UI של סביבת היעד. ה-design tokens המדויקים בסעיף 7.

> שימו לב: הפרוטוטייפ מעוצב בשפת "ברושור וינאי קלאסי" — נייר שמנת, בורדו עמוק, אוקר זהוב, גופן Calibri. זו שפת עיצוב מכוונת ויש לשמר אותה.

## 4. מסכים ותצוגות (Screens / Views)

האפליקציה היא מסגרת טלפון יחידה עם **5 לשוניות תחתונות (Tab Bar)** + שכבות-על (overlays). להלן כל מסך:

### 4.0 מסך פתיחה / ברכה (Welcome — overlay מלא בכניסה הראשונה)
- **מטרה:** רגע ה"וואו" של המתנה. מוצג רק בכניסה הראשונה (נשמר ב-storage).
- **פריסה:** מסך מלא, רקע גרדיאנט בורדו רדיאלי (`#8c2333 → #5d141d → #3c0c13`), טקסט במרכז, קונפטי מונפש נופל מלמעלה.
- **רכיבים:**
  - תג עגול "VIE · 2026" (מסגרת לבנה שקופה)
  - "יום הולדת שמח" (eyebrow, אוקר בהיר `#d8c08a`, letter-spacing .26em)
  - "שלומי" — שם המשתמש, 62px, Calibri bold, לבן שמנת `#fdf6ea`
  - אימוג'י עוגה 🎂 46px
  - פסקת ברכה אישית (16.5px) חתומה "באהבה, יפעת 💛"
  - כפתור "השמע את הברכה בקול של יפעת" — מנגן `welcome.m4a` (ראו 4.6 קול)
  - כפתור CTA ראשי "יאללה, מתחילים לטייל ✈️" (רקע אוקר `#b0832f`) + "דלג"
- **התנהגות:** לחיצה על CTA או דלג → המסך נעלם (fade+slide up .5s), נשמר `seen_welcome=true`, ואז toast "ברוך הבא, שלומי".

### 4.1 בית (Home tab)
- **מטרה:** מסך הפתיחה היומי — "מה עושים היום".
- **פריסה:** עמודה גלילה, padding 4px 20px 96px.
- **רכיבים (מלמעלה למטה):**
  1. כותרת מסך: kicker "בוקר טוב, שלומי 👋" + h1 "מה עושים היום"
  2. **כרטיס טיזר** (gradient בורדו) — אימוג'י גדול, כותרת, תת-כותרת, נקודות אינדיקציה. **מתחלף אוטומטית כל 6 שניות** ובלחיצה. תוכן הטיזרים ב-`data.js → teasers[]`.
  3. **כרטיס "יום ההגעה"** (מוצג רק ביום 1 ואם הוזן זמן נחיתה) — ציר זמן: נחיתה → דרכונים → הסעה → צ'ק-אין → יציאה לסיור, עם שעות מחושבות. ראו לוגיקת `arrivalPlan()` בסעיף 6.
  4. **כרטיס "מה עושים היום"** — תג יום צבעוני, התחנה הראשונה (pin ממוספר), שעת יציאה, וכפתורי "נווט אליי" / "פרטים".
  5. כרטיס "מצב סיור חי" — כפתור "נסה" שמדגים את מנוע ההמלצות.
  6. **רשימת תחנות היום** — שורות עם pin ממוספר, שם, מחיר, שעה, chevron. badge ירוק ✓ אם בוקר.
  7. בורר ימים (1–4)
  8. כרטיס "מתנה מיפעת" — כפתור לנגן שוב את הברכה.

### 4.2 מפה (Map tab)
- **מטרה:** מפה חיה של וינה והסביבה עם כל הנקודות.
- **פריסה:** מפה במסך מלא (Leaflet), עם שכבות-על צפות.
- **רכיבים:**
  - **מפת Leaflet** (אריחי CARTO Voyager). ראו 4.7 למפתח מפות.
  - **מסנן ימים** (למעלה): "הכל / יום 1 / יום 2 / יום 3 / יום 4" — מסנן מרקרים וקווי מסלול.
  - **3 כפתורי שכבות** (למטה מימין): 🍽️ אוכל מקומי · 🏛️ אטרקציות · 📍 איפה הייתי. כל אחד מדליק/מכבה שכבת מרקרים.
  - **כפתור מיקום** (למטה משמאל) — ממרכז על מיקום המשתמש.
  - **מרקרים ממוספרים** (11 אתרי מסלול) — צורת טיפה מסובבת 45°, צבע לפי יום, מספר במרכז.
  - **מרקרי אוכל** (9) ו**אטרקציות** (8) — עיגול עם אימוג'י.
  - **מרקר מיקום משתמש** — נקודה כחולה `#2a6cd6` עם הילה פועמת.
  - **מרקר מלון** 🏨 (אם הוזן שם מלון).
  - **קווי מסלול** מקווקווים בצבע היום בין נקודות.
- **התנהגות:** לחיצה על מרקר אתר → Sheet פרטים (4.8). לחיצה על אוכל/אטרקציה → Sheet מתאים.

### 4.3 מסלול (Trip tab)
- **מטרה:** המסלול המלא, 4 ימים, 11 אתרים.
- **רכיבים:** לכל יום — כותרת (תג יום + שם + תאריך אם שובץ), כרטיס עם רשימת תחנות (pin, שם, שעות, מחיר), וכפתור "צא למסלול היום".

### 4.4 יומן (Journal tab)
- **מטרה:** ציר זמן של המסע, דירוגים, תמונות וסרטונים.
- **רכיבים:**
  - כרטיס **חיבור Google Photos / Drive** (שני כפתורי connect עם אייקון Google).
  - לכל יום: **ציר זמן (timeline)** של מקומות שביקרת בהם (j-entry עם dot, שם, כוכבים, תאריך, הערה), **רשת תמונות** (`<image-slot>` × 4), ו**כרטיס "סרטון רגעים"** עם כפתורי "צפה" + "שתף".
  - **כרטיס הקדשה אחרון** (tribute) — "סרטון יום ההולדת של שלומי", עם נעילה ("ייפתח שעה אחרי הנחיתה בחזרה") וכפתור תצוגה מקדימה.

### 4.5 תכנון (Plan tab)
- **מטרה:** הזנת כל הפרטים האמיתיים של הטיול.
- **רכיבים (כל שדה נשמר ב-storage):**
  - **טיסות:** תאריך+שעת המראה, תאריך+שעת נחיתה בוינה, תאריך+שעת נחיתה בחזרה.
  - **מלון:** שם מלון (text), אזור/מיקום (select מתוך `hotelAreas[]`).
  - **הסעה מהשדה:** בורר שיטה (רכבת CAT / מונית שהוזמנה / שאטל / רכבת S7). אם "מונית" — נחשפים שדות: חברת מונית, שעת איסוף, מס' הזמנה/קוד. + שדה "פרטים נוספים" חופשי + כפתור "שמור פרטי הגעה".
  - **התראות והעדפות:** 4 toggles — תזכורות, פינות "הידעת?", הקול של יפעת, מיקום חי (GPS).
  - כפתור "הדגם איך עובדות ההתראות".
  - **רשימת תזכורות** מחושבת לפי התאריכים.

### 4.6 שכבות-על (Overlays)
- **Sheet פרטי אתר** (bottom sheet) — נפתח מלמטה. מכיל: pin+שם+שם אנגלי, שעות/הגעה/מחיר, פינת "הידעת?" עם קישור ויקיפדיה + כפתור קול, תיבת דירוג (5 כוכבים + הערה), כפתורי "נווט אליי ברגל" + "סגור".
- **Sheet אוכל / אטרקציה** — דומה, עם "למה דווקא כאן?" וקישור.
- **מצב הליכה (Walk overlay)** — מסך מלא: מפת ניווט, באנר הוראת ניווט עליון ("פנה ימינה", מרחק), כרטיס יעד תחתון (שם, ETA, מרחק שנותר, שעת הגעה), כפתור "הגעתי ליעד". מדמה תנועה לאורך קו אל היעד וסורק קרבה כל 2.5 שנ'.
- **חלון המלצה חיה (rec-pop)** — קופץ כשמתקרבים למקום מומלץ: אימוג'י, שם, מרחק, ההערה, **"סטייה מהמסלול: +X דק'"**, וכפתורי "הוסף לעצירה" / "תודה, ממשיכים".
- **נגן סרטון רגעים (player)** — סליידשואו בסגנון Story: בר התקדמות עליון, פריימים מתחלפים (תמונות/אימוג'י) עם כיתוב, כפתור שיתוף.
- **טוסטים (toasts)** — התראות קופצות למעלה.

### 4.7 "הקול של יפעת" (מנגנון קול)
כל כפתורי הקול מנגנים קובץ שמע מהנתיב `app/voice/<key>.m4a`:
- `welcome.m4a` — ברכת הפתיחה
- `site1.m4a` … `site11.m4a` — אחד לכל אתר
אם הקובץ לא קיים — מוצג toast עדין ("נקליט ונשבץ בקרוב"). **המשתמש מקליט ומוסיף את הקבצים — אין צורך בפיתוח.**

## 5. אינטראקציות והתנהגות (Interactions & Behavior)

- **ניווט בין לשוניות:** מחליף מסך פעיל (`display:block`), נשמר `tab` ב-storage (אפליקציה זוכרת איפה היית).
- **טיזרים:** מתחלפים אוטומטית כל 6 שנ' כשמסך הבית פעיל.
- **מצב הליכה:** הוראות ניווט מתחלפות כל 3.8 שנ'; מרקר המשתמש זז לאורך קו ל-16 שנ'; ETA ומרחק יורדים בזמן אמת.
- **מנוע קרבה:** כל 2.5 שנ' במצב הליכה — מחשב מרחק Haversine לכל מקום אוכל/אטרקציה; אם < 200 מ' וטרם הוצע — מקפיץ המלצה עם חישוב סטייה (`(dist_to_place + dist_place_to_dest − dist_direct) / 80 מ' לדקה`).
- **דירוג:** לחיצה על כוכב שומרת דירוג + מוסיפה אוטומטית ליומן (אם טרם נוסף) + toast.
- **הוספת עצירה ספונטנית:** "הוסף לעצירה" מ-rec-pop → נכנס ליומן עם תג "עצירה ספונטנית".
- **אנימציות:** confetti נופל (loop), הילת מיקום פועמת (loop), bottom sheets מחליקים (cubic-bezier .22,1,.36,1), player frames fade .6s.
- **Persistence:** כל המצב נשמר ב-`localStorage` (בנייטיב → אחסון מקומי / Firestore). **חשוב:** במעבר לנייטיב יש להחליף את שכבת ה-storage (ראו 6).

## 6. ניהול מצב (State Management)

כל המצב מנוהל דרך wrapper מעל `localStorage` (פונקציות `LS.get/set` ב-`app.js`). מפתחות:

| מפתח | תוכן |
|---|---|
| `seen_welcome` | boolean — האם הברכה כבר הוצגה |
| `tab` | string — הלשונית הנוכחית |
| `current_day` | number — היום הנבחר במסך הבית |
| `settings` | object — טיסות, מלון, הסעה, GPS (ראו מבנה ב-`settings()`) |
| `start_date` | string — תאריך נחיתה בוינה (נגזר מ-settings, מניע את שיבוץ הימים) |
| `ratings` | object — `{ <key>: { stars, note } }` לכל אתר/מקום |
| `journal` | array — רשומות ביקור `{ key, name, type, lat, lng, day, ts, spontaneous }` |
| `connect` | object — `{ drive, photos }` סטטוס חיבור |
| `reminders_on`, `dyk_on`, `voice_on` | booleans — העדפות |

**פונקציות לוגיקה מרכזיות (ב-`app.js`):**
- `arrivalPlan()` — בונה את ציר הזמן של יום ההגעה מזמן הנחיתה + שיטת ההסעה (`TRANSFER_MIN` = זמני הסעה לפי שיטה) + 45 דק' דרכונים/מזוודות + 30 צ'ק-אין + 30 רענון.
- `proximityScan(pos, dest)` — מנוע הקרבה.
- `haversine(a, b)` — מרחק גאוגרפי במטרים.
- `buildReminders()` — בונה תזכורות לפי התאריכים.

> **למפתח:** בנייטיב, החליפו את `LS` ב-Firestore (סנכרון ענן) או Capacitor Preferences (מקומי). כל הקריאות עוברות דרך אובייקט אחד — זה נקודת חיבור יחידה.

## 8. נכסים (Assets)

- **מפות:** אריחי CARTO Voyager (`https://{s}.basemaps.cartocdn.com/rastertiles/voyager/...`) — חינם לשימוש לא-מסחרי בנפח נמוך. **לאפליקציה אמיתית: עברו ל-Mapbox / Google Maps SDK** (ראו 9).

## 9. ה-Stack המומלץ לאפליקציה אמיתית (Recommended Implementation)

המלצה ל-stack שמאפשר לעטוף את הקוד הקיים עם מינימום כתיבה מחדש ולפתוח את כל היכולות:

| שכבה | טכנולוגיה | למה |
|---|---|---|
| מעטפת נייטיב | **Capacitor** (Ionic) | עוטף את ה-Web app הקיים לאפליקציית iOS/Android אמיתית, כמעט אפס כתיבה מחדש |
| UI | לשמר HTML/CSS הקיים, או לפרק ל-**React/Vue** רכיבים | הפרוטוטייפ כבר ב-vanilla JS מודולרי |
| מפות | **Mapbox GL** או **Google Maps SDK** | אריחים יציבים, ניווט אמיתי, offline |
| Backend | **Firebase** (Firestore + Storage + Cloud Functions + FCM) | מנוהל, מהיר להקמה |
| תמונות | **Google Photos Library API** + Photos picker | סנכרון לפי תאריך/מיקום |
| התראות | **Capacitor Local Notifications** + **FCM Push** | תזכורות מקומיות + שליחה מהשרת |
| מיקום | **Capacitor Geolocation** (+ background mode) | GPS חי למנוע ההמלצות |
| וידאו | **Shotstack / ffmpeg** ב-Cloud Function | הפקת סרטוני הרגעים |

## 11. צעדים ראשונים למפתח (Getting Started)

1. **פתח חשבונות:** Apple Developer ($99/שנה), Google Play ($25 חד-פעמי), Firebase, Google Cloud project.
2. **הקם Capacitor:** `npm init @capacitor/app`, העתק את תוכן `prototype/` ל-`www/`, `npx cap add ios android`.
3. **החלף את שכבת ה-storage** (`LS` ב-`app.js`) ל-Capacitor Preferences / Firestore — נקודת חיבור יחידה.
4. **החלף אריחי מפה** ל-Mapbox/Google SDK.
5. **חבר Geolocation** → הזרם מיקום אמיתי ל-`proximityScan()`.
6. **הקם Firebase** — Firestore (נתונים), Storage (מדיה), Functions (תזמון/וידאו), FCM (Push).
7. **חבר Google Photos API** למסך החיבור ביומן.
8. **בנה את הפקת הווידאו** ב-Cloud Function.
9. בדיקה על מכשיר אמיתי → העלאה לחנויות.
